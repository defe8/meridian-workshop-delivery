Inventory Dashboard — Architecture Overview

Prepared for Meridian Components IT · April 27, 2026

1. Purpose

This document captures the current state of Meridian's inventory dashboard application as of the engagement handover. It is intended as orientation for Meridian IT — to understand what is running, where it lives, and what surfaces will need to be touched for ongoing change.

2. System at a glance

Browser

Vue 3 SPA

  • Vite dev server (port 3000)
  • Vue Router 4 (history mode)
  • Composition API + Composables
  • Axios HTTP client
  • Custom i18n (EN / IT / JA)
Application

FastAPI Service

  • Python 3.12 / Uvicorn (port 8001)
  • Pydantic models
  • CORS open to localhost
  • ~25 REST endpoints under /api/*
  • Stateless request handlers
Data

Mock Data Layer

  • JSON files in server/data/
  • Loaded into memory at startup via mock_data.py
  • Writes (Tasks, POs) are in-memory only
  • No database. No persistence. (See §6.)

3.3 API client

All HTTP calls go through client/src/api.js, which wraps Axios around /api/*. Reports.vue was migrated from a hardcoded URL to this client during R1 — every view now uses the same surface, which is the point at which to add auth headers, retry logic, or environment-driven base URLs.

4.2 Filter helpers

Two helpers in main.py compose every filtered endpoint: filter_by_month() (handles single month, quarter Q1-2025, or YYYY-MM:YYYY-MM range — extended in R5) and apply_filters() (warehouse, category, status). Adding a new filtered endpoint is a one-line addition once the route is declared.

4.3 Purchase order flow

POST /api/purchase-orders accepts either backlog_item_id (reactive — fix a known shortage) or sku+warehouse (proactive — restocking recommendation). Resolution logic in _resolve_item_for_po() looks up the SKU and item name from the appropriate source. Defaults are applied for fields the operator doesn't supply (delivery date = today + 14d, supplier = "TBD — Awaiting Procurement", status = Draft).

5. Cross-cutting concerns

5.1 Internationalization

Three locales shipped in client/src/locales/: en.js, it.js, ja.js. The locale dictates the display currency (USD / EUR / JPY) via useI18n.currentCurrency; utils/currency.js performs the conversion. Adding a fourth locale is two changes: drop a new file in locales/, register it in the useI18n() composable.

5.2 Filter contract

Every filtered view subscribes to useFilters() and watches the same four signals — periodFrom/To, selectedLocation, selectedCategory, selectedStatus. The serialized output is the same query-string shape across all endpoints, so a backend change (e.g. range support) only needs to be lifted once.

6. Known constraints & risks

No persistence. Tasks and POs created at runtime live in Python lists. Restarting main.py wipes them. The shape is designed to make swapping in SQLite or Postgres a one-file change (mock_data.py becomes a repository), but that work is out of scope for this engagement.
No authentication. useAuth() returns a static user. CORS is open to localhost. Before any production exposure this layer needs a real identity provider integration.
In-memory mock data is large enough to be realistic (~250 orders, ~30 SKUs across 3 warehouses) which lets the UI feel real during demos without standing up a database.
